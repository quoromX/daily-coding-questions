**Daily Coding Challenge - Day 13: Memory Calculator**

A method is a function with a receiver. In Go, methods let you attach behavior to a type.

Write a calculator that remembers its current result.

The calculator should start with a result of `0`.

Each math method should accept a variadic list of integers.

- If two or more numbers are provided, start with the first number and apply the operation to the remaining numbers.
- If one number is provided, apply the operation using the current result and that number.
- If no numbers are provided, return the current result unchanged.
- Store the answer as the new current result before returning it.

Implement these methods on `Calculator`:

- `Add(nums ...int) int`
- `Subtract(nums ...int) int`
- `Multiply(nums ...int) int`
- `Divide(nums ...int) int`
- `Current() int`
- `Reset()`

For division, use integer division.

If division by `0` would happen, skip that `0` and continue with the next number.

Use pointer receiver methods for any method that changes the calculator.

**Examples**
```go
calculator := Calculator{}
```

`calculator.Add(1, 2)` returns `3`
*Explanation:*
`1 + 2 = 3`, and `3` becomes the current result.

`calculator.Add(3)` returns `6`
*Explanation:*
Only one number was provided, so the calculator uses the current result: `3 + 3 = 6`.

`calculator.Multiply(2)` returns `12`
*Explanation:*
The current result is `6`, so `6 * 2 = 12`.

**Test Cases**
