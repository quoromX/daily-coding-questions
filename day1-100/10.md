**Daily Coding Challenge - Day 10: Range Report**

The `range` keyword lets you loop over built-in types like slices, arrays, maps, and strings.

Write a function that builds a small report using `range` over different built-in types.

The function `rangeReport` should accept three parameters: a slice of integers called `nums`, a map of string keys to integer values called `scores`, and a string called `word`.

It should return a `Report` with these fields:

- `EvenIndexSum`: the sum of the numbers in `nums` that are stored at even indexes.
- `ScoreTotal`: the sum of all values in the `scores` map.
- `RuneCount`: the number of runes in `word`.

Use `range` to loop over each input.

Remember that when ranging over a string, Go gives you the byte index and the rune. For this challenge, count the runes, not the bytes.

**Examples**
`rangeReport([]int{2, 3, 4, 5}, map[string]int{"a": 10, "b": 20}, "go")` returns `Report{EvenIndexSum: 6, ScoreTotal: 30, RuneCount: 2}`
*Explanation:*
The even indexes in `nums` are `0` and `2`, so `2 + 4 = 6`. The map values add up to `30`. The word `"go"` has `2` runes.

`rangeReport([]int{5, 10, 15}, map[string]int{"x": 1, "y": 2, "z": 3}, "gø")` returns `Report{EvenIndexSum: 20, ScoreTotal: 6, RuneCount: 2}`
*Explanation:*
The even indexes are `0` and `2`, so `5 + 15 = 20`. The map values add up to `6`. The word `"gø"` has `2` runes, even though it uses more than `2` bytes.

`rangeReport([]int{}, map[string]int{}, "")` returns `Report{EvenIndexSum: 0, ScoreTotal: 0, RuneCount: 0}`
*Explanation:*
Empty built-in types have nothing to loop over.
