**Daily Coding Challenge - Day 17: First Duplicate**

Generics let you write functions that work with different types while still keeping type safety.

Write a generic function that finds the first duplicate value in a slice.

The function `firstDuplicate` should accept a slice of any comparable type.

Scan the slice from left to right.

Return the first value that appears for a second time, along with `true`.

If there is no duplicate, return the zero value for that type and `false`.

**Examples**
`firstDuplicate([]int{1, 2, 3, 2, 1})` returns `2, true`
*Explanation:*
`2` is the first value that appears for a second time.

`firstDuplicate([]string{"go", "js", "go", "js"})` returns `"go", true`
*Explanation:*
`go` appears for a second time before `js` does.

`firstDuplicate([]int{1, 2, 3})` returns `0, false`
*Explanation:*
There are no duplicate values, so the zero value for `int` is returned.

**Test Cases**
`firstDuplicate([]int{})` = `0, false`
`firstDuplicate([]int{1, 2, 3, 2, 1})` = `2, true`
`firstDuplicate([]int{1, 2, 3})` = `0, false`
`firstDuplicate([]string{"go", "js", "go", "js"})` = `"go", true`
`firstDuplicate([]string{"a", "b", "c"})` = `"", false`
`firstDuplicate([]bool{false, true, false})` = `false, true`
