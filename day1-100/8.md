**Daily Coding Challenge - Day 8: Login Attempt Tracker**

A closure is a function that remembers values from the place where it was created, even after that outer function has finished running.

Write a function that returns another function for tracking failed login attempts.

The outer function `makeLoginTracker` should accept one integer parameter, `maxAttempts`.

It should return a function that accepts one boolean parameter, `success`.

Each time the returned function is called:

- If `success` is `true`, reset the failed attempt count to `0` and return `true`.
- If `success` is `false`, increase the failed attempt count by `1`.
- Return `true` while the number of failed attempts is less than or equal to `maxAttempts`.
- Return `false` once the failed attempt count becomes greater than `maxAttempts`.

**Examples**
```go
tracker := makeLoginTracker(3)
```

`tracker(false)` returns `true`
*Explanation:*
This is the first failed attempt. The account is still allowed.

`tracker(false)` returns `true`
*Explanation:*
This is the second failed attempt. The account is still allowed.

`tracker(false)` returns `true`
*Explanation:*
This is the third failed attempt. The account is still allowed.

`tracker(false)` returns `false`
*Explanation:*
This is the fourth failed attempt, which is greater than `maxAttempts`.

`tracker(true)` returns `true`
*Explanation:*
A successful login resets the failed attempt count.
