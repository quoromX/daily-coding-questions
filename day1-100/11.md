**Daily Coding Challenge - Day 11: Pointer Balance Update**

A pointer stores the memory address of a value. In Go, pointers let a function update the original value instead of only changing a copy.

Write a function that updates a bank account balance using a pointer.

The function `applyTransaction` should accept two parameters: a pointer to an `Account` called `account`, and an integer called `amount`.

It should return `true` if the transaction was applied, and `false` if it was rejected.

Each account has a `Balance` and an `Active` field.

- If `account` is `nil`, return `false`.
- If `account.Active` is `false`, return `false`.
- If applying `amount` would make the balance less than `0`, return `false`.
- Otherwise, update `account.Balance` by adding `amount`, then return `true`.

The function must update the original account through the pointer.

**Examples**
```go
account := Account{Balance: 100, Active: true}
```

`applyTransaction(&account, 50)` returns `true`
*Explanation:*
The account is active, so the balance becomes `150`.

`applyTransaction(&account, -40)` returns `true`
*Explanation:*
The account has enough money, so the balance becomes `110`.

`applyTransaction(&account, -200)` returns `false`
*Explanation:*
The transaction would make the balance negative, so the balance stays `110`.

**Test Cases**
```go
account := Account{Balance: 100, Active: true}
applyTransaction(&account, 50) = true
account.Balance = 150
applyTransaction(&account, -25) = true
account.Balance = 125
applyTransaction(&account, -200) = false
account.Balance = 125
```

```go
account := Account{Balance: 20, Active: false}
applyTransaction(&account, 10) = false
account.Balance = 20
```

```go
applyTransaction(nil, 10) = false
```

```go
account := Account{Balance: 0, Active: true}
applyTransaction(&account, 0) = true
account.Balance = 0
applyTransaction(&account, -1) = false
account.Balance = 0
```
