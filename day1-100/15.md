**Daily Coding Challenge - Day 15: Ordered Priority Tasks**

Go does not have a special enum keyword. Instead, you can create enum-like values with a custom type and constants.

Write a function that orders unfinished tasks by priority.

Create a `Priority` type with these constants:

- `Low`
- `Medium`
- `High`
- `Urgent`

Each `Task` has a `Title`, a `Priority`, and a `Done` field.

The function `orderedTasks` should accept a slice of `Task` values.

Return a slice containing the titles of unfinished tasks, ordered from highest priority to lowest priority.

Ignore tasks where `Done` is `true`.

If two unfinished tasks have the same priority, keep them in the same order they appeared in the input.

If all tasks are done or the slice is empty, return an empty slice.

**Examples**
`orderedTasks([]Task{{Title: "Email", Priority: Low}, {Title: "Fix bug", Priority: High}, {Title: "Deploy", Priority: Urgent}})` returns `[]string{"Deploy", "Fix bug", "Email"}`
*Explanation:*
`Urgent` comes before `High`, and `High` comes before `Low`.

`orderedTasks([]Task{{Title: "A", Priority: High}, {Title: "B", Priority: High}, {Title: "C", Priority: Medium}})` returns `[]string{"A", "B", "C"}`
*Explanation:*
`A` and `B` have the same priority, so their original order is preserved.

**Test Cases**
`orderedTasks([]Task{})` = `[]string{}`
`orderedTasks([]Task{{Title: "Email", Priority: Low}, {Title: "Fix bug", Priority: High}, {Title: "Deploy", Priority: Urgent}})` = `[]string{"Deploy", "Fix bug", "Email"}`
`orderedTasks([]Task{{Title: "Deploy", Priority: Urgent, Done: true}, {Title: "Review", Priority: Medium}})` = `[]string{"Review"}`
`orderedTasks([]Task{{Title: "A", Priority: High}, {Title: "B", Priority: High}, {Title: "C", Priority: Medium}})` = `[]string{"A", "B", "C"}`
`orderedTasks([]Task{{Title: "Done", Priority: Urgent, Done: true}})` = `[]string{}`
